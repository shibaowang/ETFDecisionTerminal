# EPIC-284 Local Trial Visual Acceptance Pack

EPIC-284 adds a local visual evidence pack for the EPIC-282/283 release
candidate. It is an acceptance aid only; it does not add business behavior,
trading behavior, or automatic execution.

## Purpose

The evidence pack gives a reviewer a repo-local bundle of command logs,
process logs, CTest output, optional screenshots, and a compact JSON summary.
The default output location is:

```text
.demo/local_trial_rc/evidence/
```

The pack is generated by:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\local_trial\Invoke-ETFDTLocalTrialVisualEvidence.ps1
```

## Screenshot Rules

Screenshot capture depends on the local Windows desktop session. If a desktop
window cannot be found or screen capture is unavailable, the script must record
`screenshotAvailable=false` and a clear fallback reason. It must not fabricate
screenshots or claim that a human manually confirmed the UI.

Generated screenshots, when available, are reviewer evidence only. They still
require human review before release acceptance.

After EPIC-285, screenshot review should focus on Chinese readability for the
Dashboard MVP:

- page title shows `ShellAccounting 本地试用看板`
- module titles include `Excel/VBA 导入`, `持仓/现金重算`, `行情数据`,
  `策略建议`, `交易草案`, and `场外 A/C 多通道`
- draft safety text says `这是内部草案，不是订单，不会提交券商。`
- market data safety text says `默认使用样例数据/禁用行情源，不会自动联网。`
- technical IDs may remain English in diagnostics or evidence, but visible
  trial workflow instructions should be understandable in Chinese

Expected screenshot names are:

- `dashboard_startup.png`
- `dashboard_shellaccounting.png`
- `dashboard_import_panel.png`
- `dashboard_market_panel.png`
- `dashboard_strategy_panel.png`
- `dashboard_tradedraft_panel.png`
- `dashboard_otcmap_panel.png`

## Evidence Files

The evidence directory should contain:

- `commands.log`
- `ctest.log`
- `process.log`
- optional `dashboard_*.png` screenshots
- `visual_acceptance_report.md`
- `visual_acceptance_evidence.json`

The script starts only the repo-local trial workspace, repo-local demo DB, local
DataService process, and local Shell process. Cleanup is always attempted and
uses the repo-local pid files created by the local trial scripts.

## Boundaries

EPIC-284 does not authorize:

- production database access or mutation
- migrations
- external downloads
- administrator or system setup
- live provider access by default
- broker, credential, private endpoint, real order placement, or automatic
  trading work
- new QML business behavior
- Strategy, Replay, TradeDraft, OTCMap, or MarketData semantic changes

The visual evidence pack is a semi-automated acceptance helper. It is not a
replacement for final human review.
